# GitHub Bulk Writer Integration

This module is isolated at `src/lib/github-bulk-writer` and is not wired into the app yet.

## What changes

The current `src/lib/github.ts` flow creates a repository, then calls `createOrUpdateFileContents` once per file. GitHub turns that into one commit per file.

The bulk writer uses Git data endpoints instead:

1. Create the repository with `auto_init: false`.
2. Create one blob for each generated file.
3. Create one tree that references every blob.
4. Create one root commit with that tree.
5. Create `refs/heads/main` or the requested default branch at that commit.
6. Update the repository default branch when the requested branch is not GitHub's default.

For a brand-new empty repository, the first branch ref must be created with `git.createRef`. `git.updateRef` only works after a ref exists, so the module keeps a non-forced `updateRef` fallback for unexpected existing refs.

## Safe replacement

Do not make this change while other work is editing `src/lib/github.ts`. When the integration window is clear, replace the current per-file loop with the wrapper below:

```ts
import { createRepositoryWithInitialCommit } from "@/lib/github-bulk-writer";

import type { GeneratedFile } from "@/lib/stacks";

export async function createRepositoryWithFiles({
  files,
  projectName,
  token,
}: {
  files: GeneratedFile[];
  projectName: string;
  token: string;
}) {
  const result = await createRepositoryWithInitialCommit({
    githubToken: token,
    repoName: projectName,
    description: "Generated by AgentStack.",
    private: false,
    files,
  });

  return result.repositoryUrl;
}
```

Keep `src/app/api/generate/route.ts` unchanged unless the route needs different user-facing error text.

## Error mapping

The writer throws typed, user-safe errors:

```ts
import {
  GitHubRateLimitError,
  GitHubRepoAlreadyExistsError,
  GitHubUnauthorizedError,
  GitHubValidationError,
} from "@/lib/github-bulk-writer";

function getGitHubErrorMessage(error: unknown) {
  if (error instanceof GitHubUnauthorizedError) {
    return "GitHub could not use this token. Reconnect GitHub and try again.";
  }

  if (error instanceof GitHubRepoAlreadyExistsError) {
    return "A repository with this name already exists on GitHub.";
  }

  if (error instanceof GitHubRateLimitError) {
    return "GitHub is rate limiting requests right now. Try again in a few minutes.";
  }

  if (error instanceof GitHubValidationError) {
    return "GitHub could not create this repository. Check the repo name and generated files.";
  }

  return "GitHub could not create this repository.";
}
```

## Path safety

No generated-file safety package exists in this project today, so the writer uses a local guard. It rejects empty paths, absolute paths, traversal segments, Windows separators, control characters, duplicate paths, and `.git` writes.

If a shared generated-file safety module is added later, replace `validateGitHubBulkWriterPath` with that validator and keep the same typed error behavior.

## Checks before rollout

- Run `npm run typecheck`.
- Create a test repository with at least two files and confirm GitHub shows one initial commit.
- Confirm existing route responses still map errors without leaking token or API details.
- Decide whether `owner` should stay unset for user repositories or point at an organization login. Non-authenticated-user owners are treated as organizations.
